# Strukturierte Kommunikation und „System als Manager“

Wir gestalten Kommunikation so, dass sie Produktivität und Lernen maximiert:

- So wenig informelle Ad‑hoc‑Kommunikation wie möglich (E‑Mail, Chat, spontane Nachrichten außerhalb der vorgesehenen Slots).
- Nutzung standardisierter Prozesse/Portale (E‑Mail-System, Launchpad-Workflows, GitHub, Kalender), damit Informationen ohne Ping‑Pong im System landen.
- Ziel: weniger Bürokratie, mehr Fokuszeit, mehr echte Lehre/Coaching.

## System als Manager

Anstatt menschliche Manager mit Mikromanagement einzusetzen, programmieren wir unsere Arbeitsabläufe. Das Launchpad orchestriert u. a.:

- Termin-Erinnerungen und Networking‑Slots (z. B. 10:14) mit nicht‑blockierendem Hinweiston.
- Automatische Kanalwechsel in Discord zum offiziellen Meetingstart und die Rückkehr danach.
- 7‑Minuten‑Supportslots, die per Klick angefragt werden und automatisch abgewickelt werden (inkl. Aufnahme für Lernmaterial).

Das reduziert Verwaltungsaufwand, verhindert unnötige Unterbrechungen und erhöht die Qualität der gemeinsamen Zeiten.

Weitere Details in der Wissensbasis unter Kommunikation sowie im Launchpad‑Repository (Konzepte/Architektur).
